CCycDB-Usage and Examples

A guide to analyzing metagenomic data with CCycDB

Step 0. Prepare your files

Input files supports 3 type of files

  • Read-based

    • Raw reads (clean reads) or Merged Reads (Forward and reverse reads were merged into longer sequences by the program e.g. PEAR)

  • Assembly-based

    • Contigs generated through metagenome assmebly by (e.g., MEGAHIT, MetaSPAdes, SPAdes).

  • Tabular Files

    • BLAST table (delimited with "\t") generated through sequences similarity searching tools (e.g., BLAST, USEARCH, DIAMOND)

[Options:]

-situation the situation for input files (read-based|assembly-based|tabular)
-wd The directory may or may not exist.
-od File type. Specify the extensions of your input files (fastq, fastq.gz, fasta, fasta.gz, fq, fq.gz, fa, fa.gz) or (faa, fna). Please make sure that the filetype is correct according to the tool selected by -m. E.g., if -m usearch, the filetype supports (fastq|fasta); if -m blast, the filetype supports fasta.
-m Database searching program you plan to use (diamond|usearch|blast)
-f Specify the extensions of your sequence files (fastq, fastq.gz, fasta, fasta.gz, fq, fq.gz, fa, fa.gz) or (faa, fna). Please make sure that the filetype is correct according to the tool selected by -m. E.g., if -m usearch, the filetype supports (fastq|fasta); if -m blast, the filetype supports fasta.
-s Sequence type (nucl|prot)
-tpm (0|1)(default: 0). "1" need $sample.tpm exist in the work directory
-norm (0|1).0: don`t need random sampling; 1: need random sampling.[-situation assembly-based] is a prerequisite for this parameter
-rs the number of sequences for random subsampling. (default: the lowest number of sequences).Note: [-norm 1] is a prerequisite for this parameter.
-thread number of CPUs (default: 2)
